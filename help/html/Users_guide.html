Specify the model

The first step to optimizing of model parameters or corresponding measurement conditions with the Optimal Experimental Design Toolbox is to specify the model function. This is done by providing an object that implements the model interface, which prescribes methods for the model function and its first and second derivative with respect to the model parameters.

model_object = model_class();

If the model function is given in an explicit form you can use the model_explicit class. The necessary function values and derivatives will then be calculated automatically.

model_object = model_explicit(model_function, parameter, dependent_variable);

If the model function is given as an initial value problem you can use the model_ivp class. There only the initial value problem must be specified. The necessary function values and derivatives will then be calculated automatically.

model_object = model_ivp(dependent_variable, independent_variable, parameter, ...
    differential_equation, integration_interval, starting_point);

If you are using the model_ivp class you can obtain a speedup by calculating the necessary derivations of the model in parallel. To achieve that you only have to open a matlabpool before the calulations.

matlabpool open number_of_cores

If you have an arbitrary implementation of the model function without the derivatives with respect to the parameters you can implement the model_fd interface instead of the model interface. This provides the first and second derivatives with respect to the parameters by finite differences approximations.

Creating and configuring an solver object

The next step is to create and configure an solver object. First of all you have to create an solver object via the constructor.

solver_object = solver(model_object, ...
    parameter_estimation, measuring_points, corresponding_variances);

Then you have to pass the previously created model object presenting your model function to your solver object. This can be done via the constructor or the set_model method.

solver_object.set_model(model_object);

Further an approximation of the parameter values must be specified via the constructor or the set_initial_parameter_estimation method. This can arise for example from experience or previous measurements.

solver_object.set_initial_parameter_estimation(parameter_estimation);

Next you have to specify possible measurements and there corresponding variances of the measurement errors. This can be done via the constructor and the set_possible_measurements method.

solver_object.set_possible_measurements(measuring_points, ...
    corresponding_variances);

If you already have done previous measurements, this can be included. The measurements and there corresponding variances of the measurement errors can be set via the set_accomplished_measurements method.

solver_object.set_accomplished_measurements(measuring_points, ...
    corresponding_variances, measuring_results);

Optimizing measurements

Now you can calculate the optimal measurements by using the get_optimized_weightings method of your solver object. The result of the get_optimized_weightings method is a weighting 0 or 1 for each measurements. 1 means the measurement has been chosen and 0 means the measurement has not been chosen. You can pass linear constraints to the measurements. Thereby, for example, a restriction to the cost of the measurements can be achieved.

weightings = solver_object.get_optimized_weightings(A, b);

A common case is that the number of selected measurements should be restricted. This can be achieved directly as follows.

weightings = solver_object.get_optimized_weightings(maximal);

Quality of measurements

You can also calculate the quality of a set of measurements with the get_quality method. This allows, for example, to calculate the quality resulting by omitting some measurements.

quality = solver_object.get_quality(weightings);

Optimize model parameters

If you have done previous measurements you can optimize the model parameters with your measurement results. For that you can use the get_optimal_parameters method.

parameter = solver_object.get_optimal_parameters(lower_bound, upper_bound);

Changing the options

The options for the optimization of the experimental design are stored in an solver_options object. To change these options you have to pass an an solver_options with your desired options to your solver object.

solver_object.set_options(solver_options);

At the instantiation of your solver_options object you can pass your desired (non-default) options to the constructor.

solver_options_object = solver_options(...
    'option1', value1, 'option2', value2, ...);

It is also possible to set your desired options after the instantiation of your solver_options object with the set_option method.

solver_options_object.set_option('option1', value1);
solver_options_object.set_option('option2', value2);

The possible options for the experimental design are

  • 'parameter_estimation': whether a parameter estimation should be performed befor the optimal design estimation or not (possible values: 'yes', 'no', default: 'no')
  • 'estimation_method': the method of the estimation of the quality of the experimental design (possible values: 'point', 'region', default: 'region')
  • 'alpha': the confidence level for the region estimation (possible value: a scalar with 0 < alpha < 1, default: 0.95)
  • 'debug': the level of debug information to be displayed (possible values: a non-negative integer, default: 0 (no debug informations))
  • 'criterion': the criterion for the quality of an experimental design (possible values: an object of the criterion class, default: a criterion_A object)
  • 'scale_covariance_matrix': whether the covariance matrix should be scaled or not (possible values: 'yes', 'no', default: 'yes')
  • 'solver_edo_options': the solver to be used to solve the experimental design optimization problem (possible values: an object of the solver_edo_options class, default: a default solver_edo_options object)

The options for the solver of the experimental design optimization problem are stored in a solver_edo_options object. They can be set via the constructor.

solver_edo_options_object = solver_edo_options('option1', value1, 'option2', value2, ...);

You can change the options also after the instantiation with the set_option method.

solver_edo_options_object.set_option('option1', value1);
solver_edo_options_object.set_option('option2', value2);

The possible options for the solver are

  • 'algorithm': the method to be used to solve the experimental design optimization problem (possible values: 'direct', 'local_sqp', default: 'local_sqp')
  • 'max_fun_evals': the number of maximal model evaluations done by the solver (possible values: a non-negative integer, default: 10^3)
  • 'max_iter': the number of maximal iterations of the solver (possible values: a non-negative integer, default: 5*10^1)
